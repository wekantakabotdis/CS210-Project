# CS210-Project

GitHub repository: https://github.com/wekantakabotdis/CS210-Project

This project builds a master Pokemon card pricing dataset for the CS 210 project.
The dataset is created from TCGCSV's public Pokemon group, product, and price endpoints.
The notebook then cleans the dataset, creates a SQLite database, runs SQL exploration,
visualizes market patterns, and trains baseline, Ridge, and Random Forest regression
models for card-price appraisal.

## Files

- `import_pokemon_dataset.py`: downloads the data and builds the master CSV
- `main.ipynb`: notebook version for database setup, SQL exploration, visualization, and modeling
- `data/pokemon_card_price_dataset.csv`: output CSV created by the Python script
- `data/pokemon_card_price_cleaned_feature_engineered.csv`: cleaned dataset used for the SQLite database and modeling
- `data/pokemon_card_price_ml_ready.csv`: one-hot encoded numeric dataset for machine learning
- `data/pokemon_cards.db`: local SQLite database created from the cleaned dataset
- `final_report.typ`: final written report source with citations
- `CS210_Pokemon_Card_Final_Report.pdf`: final written report PDF
- `requirements.txt`: Python dependencies needed to rerun the project

## Requirements

Use Python 3.12 or a recent Python 3 version. From the project folder, install the
dependencies with:

```bash
python3 -m pip install -r requirements.txt
```

## How To Run

From the `CS210-Project` folder, regenerate the raw dataset with:

```bash
python3 import_pokemon_dataset.py
```

The script will:

- fetch all Pokemon groups from TCGCSV
- fetch each group's products and prices
- keep Pokemon card rows with usable card attributes
- build one master dataset
- save the final CSV to `data/pokemon_card_price_dataset.csv`

Then execute the notebook from top to bottom:

```bash
jupyter nbconvert --to notebook --execute main.ipynb --inplace --ExecutePreprocessor.timeout=900
```

The notebook creates the cleaned dataset, the ML-ready dataset, the SQLite
database, and the final report figures before running SQL queries,
visualizations, model training, time-based cross validation, and price-band
error analysis.

## Output

After the script finishes, the master dataset will be here:

```bash
data/pokemon_card_price_dataset.csv
```

After the notebook finishes, the derived files will be here:

```bash
data/pokemon_card_price_cleaned_feature_engineered.csv
data/pokemon_card_price_ml_ready.csv
data/pokemon_cards.db
```

The final report files are:

```bash
final_report.typ
CS210_Pokemon_Card_Final_Report.pdf
```

To rebuild the report PDF from the Typst source, run:

```bash
typst compile final_report.typ CS210_Pokemon_Card_Final_Report.pdf
```

## Notes

- The import may take a little time because it walks through all groups one by one.
- The script prints progress as each group is processed.
- The raw CSV is included so the notebook can be rerun without making network requests.
- The SQLite database is generated by the notebook and can be recreated at any time.
