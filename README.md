# CS210-Project

GitHub repository: https://github.com/wekantakabotdis/CS210-Project

This project analyzes Pokemon card market prices from public TCGCSV data. It
builds a card-finish-level pricing dataset, cleans and engineers features,
stores the cleaned data in SQLite, explores market patterns with SQL and
visualizations, and trains baseline, Ridge, and Random Forest regression models
for card-price appraisal.

## Project Structure

- `import_pokemon_dataset.py`: downloads TCGCSV Pokemon groups, products, and prices, then builds the raw master CSV
- `main.ipynb`: end-to-end notebook for cleaning, feature engineering, SQLite setup, SQL exploration, visualization, modeling, and figure export
- `requirements.txt`: Python dependencies needed to rerun the importer and notebook
- `final_report.typ`: Typst source for the final written report
- `CS210_Pokemon_Card_Final_Report.pdf`: compiled final report
- `Final_CS_210_Project_Document.pdf`: earlier project document
- `Pokemon_Card_Price_Proposal.docx`: original project proposal
- `CS210-Project-submission.zip`: packaged project submission archive
- `reports/figures/`: figures generated by the notebook and used by the final report

## Data Artifacts

- `data/pokemon_card_price_dataset.csv`: raw card-price dataset created by `import_pokemon_dataset.py`
- `data/pokemon_card_price_cleaned_feature_engineered.csv`: cleaned and feature-engineered dataset used for SQL exploration and modeling
- `data/pokemon_card_price_ml_ready.csv`: one-hot encoded numeric dataset for machine learning
- `data/pokemon_cards.db`: SQLite database created by the notebook with a `pokemon_cards` table

The current included datasets contain:

- 33,112 rows in the raw dataset
- 32,433 rows in the cleaned feature-engineered dataset
- 32,433 rows in the ML-ready dataset

## Requirements

Use Python 3.12 or another recent Python 3 version. From the project folder,
install the dependencies with:

```bash
python3 -m pip install -r requirements.txt
```

## How To Run

To regenerate the raw dataset from TCGCSV, run:

```bash
python3 import_pokemon_dataset.py
```

The importer:

- fetches all Pokemon groups from TCGCSV
- fetches each group's products and prices
- joins product metadata to price rows by product ID
- keeps rows with positive market prices and usable card attributes
- saves the raw dataset to `data/pokemon_card_price_dataset.csv`

Then execute the notebook from top to bottom:

```bash
jupyter nbconvert --to notebook --execute main.ipynb --inplace --ExecutePreprocessor.timeout=900
```

The notebook reads the raw CSV, creates the cleaned and ML-ready CSVs, writes
`data/pokemon_cards.db`, runs SQL exploration, saves report figures to
`reports/figures/`, trains and evaluates the models, and performs time-based
cross validation and price-band error analysis.

## Outputs

Running the importer creates or refreshes:


Running the notebook creates or refreshes:

```bash

data/pokemon_card_price_ml_ready.csv
```

To rebuild the final report PDF from the Typst source, run:

```bash
typst compile final_report.typ CS210_Pokemon_Card_Final_Report.pdf
```

## Notes

- The raw CSV is included so the notebook can be rerun without making network requests.
- The importer may take a while because it walks through all TCGCSV Pokemon groups one by one.
- The SQLite database is generated by the notebook and can be recreated at any time.
- `.gitignore` excludes `data/*.db`, so the database may not appear in a fresh Git clone until the notebook is run.
